# 18.2 Hardening PHP (php.ini)

## Configuration sécurisée PHP

**Fichier à modifier :**

```bash
sudo nano /etc/php/8.3/apache2/php.ini
```

**Paramètres de sécurité critiques :**

```ini
; Désactiver l'affichage des erreurs (production)
display_errors = Off
display_startup_errors = Off
log_errors = On
error_log = /var/log/php/error.log

; Masquer PHP dans les headers HTTP
expose_php = Off

; Limiter les ressources
max_execution_time = 30
max_input_time = 60
memory_limit = 128M
post_max_size = 10M
upload_max_filesize = 8M

; Désactiver les fonctions dangereuses
disable_functions = exec,passthru,shell_exec,system,proc_open,popen,curl_exec,curl_multi_exec,parse_ini_file,show_source

; Restriction des fichiers ouverts
open_basedir = /var/www/html/atlastech:/tmp

; Sessions sécurisées
session.cookie_httponly = 1
session.cookie_secure = 1
session.use_strict_mode = 1
session.cookie_samesite = Strict

; Désactiver allow_url_fopen et allow_url_include
allow_url_fopen = Off
allow_url_include = Off

; Activation de filter FILTER_SANITIZE
filter.default = full_special_chars
```

**Application :**

```bash
sudo systemctl restart apache2

# Vérification
php -i | grep -E "display_errors|expose_php|disable_functions"
```
